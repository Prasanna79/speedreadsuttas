# SuttaSpeed Launch Marketing Spec v2

Date: 2026-03-07
Supersedes: `marketing/LAUNCH_MARKETING_SPEC_V1.md`

## 1. Launch Objective

Drive traffic to `https://suttaspeed.com/read/mn19/en/sujato` using:

1. one X launch post
2. one primary Reddit launch post in `r/SideProject`
3. one secondary Reddit launch post in `r/sideprojects`
4. one feedback-oriented comment for `r/webdevelopment`
5. one tester-recruitment post for `r/alphaandbetausers`

## 2. Core Story

Use the same founder story everywhere, adapted to audience:

1. I wanted to read more suttas, but long screen reading often turned into tab-hoarding
2. I found that the friction was not the texts themselves, it was attention and visual scanning
3. I built SuttaSpeed to make suttas easier to actually read through

## 3. Universal Benefit Stack

Every version should mention some subset of:

1. one-word / chunked RSVP reading
2. less eye-scanning
3. easier to sustain attention
4. adjustable speed
5. translation switching
6. immediate live demo

## 4. CTA

Primary CTA:

1. “Try MN19 here: `https://suttaspeed.com/read/mn19/en/sujato`”

Secondary CTA:

1. “If you try it, I’d love to know whether it actually helps you stay with the text.”

## 5. Channel-Specific Rules

### 5.1 X

1. keep it short
2. one clear founder story beat
3. one clear product statement
4. one direct link

### 5.2 r/SideProject

1. optimize for curiosity and traffic
2. write as a founder post, not an ad
3. let the niche be a feature, not something to apologize for

### 5.3 r/sideprojects

1. slightly more direct about launch
2. still grounded in personal motivation
3. shorter than the `r/SideProject` version

### 5.4 r/webdevelopment

1. treat as a feedback request
2. ask for UX input on typography, mobile behavior, and pacing
3. do not write it like a marketing launch

### 5.5 r/alphaandbetausers

1. treat as a tester request
2. ask for a small amount of effort
3. ask targeted questions

## 6. Avoid List

Do not use as primary launch channels without moderator approval:

1. `r/Meditation`
2. `r/theravada`
3. `r/Buddhism`
4. `r/InternetIsBeautiful`

## 7. Content Deliverables

1. `marketing/X_LAUNCH_POST.md`
2. `marketing/REDDIT_SUBREDDITS.md`
3. `marketing/reddit/r_SideProject_post.md`
4. `marketing/reddit/r_sideprojects_post.md`
5. `marketing/reddit/r_webdevelopment_feedback_comment.md`
6. `marketing/reddit/r_alphaandbetausers_post.md`

## 8. Writing Constraints

1. no fake metrics
2. no fake urgency
3. no “revolutionary” language
4. sound like one person built something useful because he personally needed it
5. always include the direct MN19 link
